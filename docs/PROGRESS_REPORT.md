## RTD site maintenance
Read the Docs doesn't have a public CLI that allows adding a new version purely from the command line. Therefore, it is always necessary to go to the RTD web dashboard and click `Add version`. Below is GPT's answer to the question.

> Version management is mainly done in the RTD web dashboard.But there are ways to approximate it with automation, depending on what you want:
>
> 1. **How RTD versions are created**
>
> RTD detects versions automatically from your Git repo:
>
> - Branches: Any branch you push to GitHub can be built as a version.You just need to enable the branch in RTD dashboard once.
> - Tags: Any Git tag pushed to GitHub can become a version. RTD will automatically promote the highest tag to stable. Older tags show as inactive versions (unless enabled manually).
>
> âœ… Key point: RTD builds versions based on Git refs â€” no dashboard interaction is required to create the version itself if the tag or branch is pushed.
>
> 2. **Command-line â€œversion creationâ€**
>
> From the CLI you can:
>
> - Create a tag in Git:
>
> ```bash
> git tag v1.2.0
> git push origin v1.2.0
> ```
> This automatically makes v1.2.0 visible to RTD (it will appear in the Versions tab). If itâ€™s the highest tag, RTD makes it stable.
>
> - Create a branch in Git:
>
> ```bash
> git checkout -b attempt
> git push origin attempt
> ```
> RTD will eventually detect it. You may need to enable it once in the dashboard.
>
> âš ï¸ Limitation
>
> You cannot fully bypass the dashboard to activate a branch/tag if itâ€™s not the â€œlatestâ€ or â€œstableâ€ automatically.
>
> Some teams use RTD API to automate enabling versions.
>
> 3. **RTD API (optional automation)**
>
> Read the Docs has a REST API (requires a token) that allows you to:
> - List versions
> - Activate/deactivate versions
> - Trigger builds
>
> Example (Python, using requests):
>
> ```python
> import requests
>
> RTD_TOKEN = "your_api_token"
> PROJECT_SLUG = "my-project"
> VERSION = "v1.2.0"
>
> url = f"https://readthedocs.org/api/v3/projects/{PROJECT_SLUG}/versions/{VERSION}/"
>
> headers = {
>     "Authorization": f"Token {RTD_TOKEN}"
> }
>
> # Activate version
> response = requests.patch(url, headers=headers, json={"active": True})
> print(response.status_code, response.json())
> ```
> You still need the tag pushed first. The API just activates the version in RTD, so it shows up in the flyout.
